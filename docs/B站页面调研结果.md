# B站页面调研结果

更新时间：2026-07-03

## 调研范围

这份文档用于指导本项目的遮罩逻辑，不是 B 站产品说明。

调研对象包括：

- B 站 Windows 客户端的首页、搜索、历史/个人页、视频播放页、视频全屏状态。
- B 站网页端首页和搜索入口。
- 用户本轮对话提供的错误截图。
- 公开网页和公开项目中能验证的 B 站页面/客户端特征。

## 重要前提

B 站 Windows 客户端不能按普通网页 DOM 处理。

本机探测发现，客户端窗口在 UI Automation 里经常只暴露宽泛的 Chromium/Electron 容器，例如 `Chrome Legacy Window`，并不稳定暴露每个按钮、列表和卡片。因此：

- 不能只靠控件树判断页面。
- 不能只靠窗口标题判断页面。
- 需要把标题、进程、窗口尺寸、左侧导航视觉状态、右侧栏截图取样结合起来。

公开资料可以帮助确认 B 站存在桌面客户端、搜索入口和网页入口，但真正用于遮罩决策的依据必须来自当前客户端窗口的实时状态。

## B 站客户端入口与页面大类

B 站官网和下载页都提供 PC/桌面客户端入口。官网首页在当前抓取环境下提示可“下载哔哩哔哩桌面客户端”，下载中心也列出“PC客户端”。

这说明本项目面对的是一个独立桌面客户端窗口，而不是浏览器标签页。它需要 Win32 窗口识别、前台窗口判断、DWM 边界读取、托盘图标和全局快捷键处理。

客户端内主要页面可以按用户意图分为两类：

- 平台主动推荐页：首页推荐流、视频页右侧推荐。
- 用户主动目标页：搜索结果、历史记录、个人页、视频选集、合集/课程列表、全屏视频。

本项目只遮挡第一类。

## 首页推荐页

观察结果：

- 左侧通常有垂直导航。
- “首页”选中时呈 B 站粉色。
- 顶部有 B 站标识、频道入口和搜索框。
- 主区域是多列视频卡片瀑布流/网格流。
- 首页推荐内容会随刷新、滚动、登录状态变化。

遮罩结论：

- 首页推荐流是应遮挡目标。
- 顶栏、搜索框、左侧导航应保留。
- 不能用标题单独判断首页，因为 B 站客户端可能在其他页面仍显示泛标题。

当前实现：

- 首页模式下默认从窗口左侧 `80px`、顶部 `96px` 之后覆盖。
- 进入遮挡前先检查搜索状态和左侧导航选中态。
- 如果泛标题窗口顶部呈视频页深色栏和右侧深色标签栏，先按视频页处理，禁止首页白遮罩盖住播放器。
- 如果左侧粉色选中区域不在首页位置，隐藏遮罩。

## 搜索页与搜索状态

观察结果：

- 用户点击搜索按钮或输入关键词后，页面会进入搜索意图。
- 搜索结果是用户明确要求看的内容，不是首页随机推荐。
- 搜索页可能仍保留 B 站顶部框架和左侧导航，因此不能按“看起来像首页”处理。

遮罩结论：

- 搜索框聚焦时不遮挡。
- 搜索框有有效关键词时不遮挡。
- 搜索结果页不遮挡。

当前实现：

- 使用 UI Automation 尝试读取顶部搜索框。
- 使用顶部中间搜索框的粉色焦点边框作为视觉兜底。
- 忽略默认占位文本和导航词。
- 识别到搜索后保持 2 秒，避免跳转过程遮罩闪现。

## 历史记录、个人页、“我的”

观察结果：

- “我的”页左侧导航选中态在更靠下的位置。
- 历史记录是用户已看内容的记录，属于用户主动访问的数据页。
- 这些页面有时也会复用 B 站泛标题。

遮罩结论：

- 历史记录不遮挡。
- “我的”、收藏、个人中心等不遮挡。
- 不能因为页面中有视频卡片就按首页推荐遮挡。

当前实现：

- 截取窗口左侧导航区域。
- 搜索 B 站粉色选中像素。
- 如果选中中心明显低于首页区域，判定为非首页页面，隐藏遮罩。

## 视频播放页

观察结果：

- 左侧是播放器和视频内容。
- 右侧上方可能有作者、简介、互动按钮。
- 右侧中部可能出现视频选集、合集、课程列表、播放列表。
- 右侧下方通常出现推荐视频列表。
- 推荐列表位置不是固定的，会随视频类型、选集数量、窗口高度变化。

遮罩结论：

- 右侧推荐列表应遮挡。
- 作者信息、操作按钮、视频简介不遮挡。
- 视频选集、播放列表、课程列表不遮挡。
- 不能从固定 `Y` 坐标开始盖右侧栏。

当前实现：

- 先用顶部深色栏、播放器顶边和右侧深色标签栏判断泛标题窗口是否其实是视频页，避免把视频页套进首页白遮罩。
- 从右侧栏候选区域截图。
- 寻找推荐视频缩略图样式的图片区块。
- 找到后从缩略图区块附近开始遮挡。
- 找不到时不遮挡。

这个方案比固定矩形更适合 B 站右侧栏，因为它根据实际推荐缩略图出现的位置决定遮罩起点。

## 视频选集

观察结果：

- 视频选集一般是文字列表或较小的条目，不一定有明显大缩略图。
- 它位于推荐列表上方时，如果直接固定遮罩右下角，会误挡选集。
- 用户明确要求选集不能被挡。

遮罩结论：

- 选集是用户主动选择下一集的控制区域，不属于推荐流。
- 右侧推荐遮罩必须在选集下面开始。

当前实现：

- 通过图片区块检测寻找真正的推荐缩略图。
- 文字列表不会轻易触发推荐遮罩。
- 用户最新未遮挡视频页截图离线验证结果显示，推荐区识别起点为 `Y=836`，位于选集下方。

## 视频全屏

观察结果：

- 全屏视频时用户只想看视频主体。
- 右侧栏通常不存在，或不应按普通窗口布局处理。
- 全屏时显示任何遮罩都会干扰观看。

遮罩结论：

- 视频全屏状态下全部隐藏遮罩。

当前实现：

- 比较 B 站窗口边界和屏幕边界。
- 接近全屏则隐藏遮罩。

## 右侧推荐遮罩的闪烁问题

问题来源：

- 曾经为了避免截图识别到自己的遮罩，采用“隐藏遮罩 -> 截图 -> 再显示遮罩”的方式。
- 这会造成肉眼可见的周期性闪烁。

结论：

- 不能周期性隐藏遮罩做视觉探测。
- 正确做法是在遮罩显示前完成探测，并缓存结果。
- 页面、标题、窗口尺寸变化后再重新探测。

当前实现：

- 找到推荐区后缓存遮罩起点。
- 遮罩已经显示时复用缓存，不反复截图。
- 如果没有缓存且遮罩可见，先隐藏并等待正常页面状态变化后再探测，不进入闪烁循环。

## 为什么没有直接上 OpenCV

OpenCV 可以更强，但当前问题不需要立即引入它。

原因：

- 本项目只需要识别少数稳定视觉特征：左侧导航粉色选中态、右侧推荐缩略图区块。
- .NET `System.Drawing.Bitmap` 已能完成截图和像素取样。
- 引入 OpenCV 会增加依赖、部署复杂度和开机自启失败风险。

当前策略：

- 先用轻量截图取样解决明确问题。
- 如果后续 B 站改版导致缩略图识别不稳，再考虑 OpenCV 模板匹配或更复杂的图像分割。

## 当前规则汇总

应遮挡：

- 首页推荐流。
- 视频页右侧推荐视频列表。

不应遮挡：

- 搜索框、搜索结果页。
- 历史记录。
- “我的”、收藏、个人中心。
- 视频选集、合集、课程列表。
- 视频全屏。
- B 站客户端不在前台。
- 无法确认推荐区位置的状态。

## 后续可改进点

- 为右侧推荐检测增加更多截图回归样例。
- 将关键阈值集中成配置项，并在 README 中说明。
- 如果客户端 UI Automation 暴露更多稳定信息，可优先使用控件语义，再 fallback 到视觉探测。
- 若 B 站右侧栏改版导致图片区块判断失效，再评估 OpenCV。
