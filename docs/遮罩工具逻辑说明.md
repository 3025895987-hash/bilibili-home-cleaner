# B站遮罩工具逻辑说明

更新时间：2026-07-03

## 目标

本工具只处理两类“平台主动推荐”：

- B 站客户端首页推荐流。
- 视频播放页右侧的推荐视频列表。

以下页面或状态不应遮挡：

- 搜索框聚焦、搜索词输入后、搜索结果页。
- “我的”、历史记录、收藏等用户主动访问的数据页。
- 视频全屏播放。
- 视频选集、课程列表、播放列表等用户正在按顺序观看的内容。
- B 站窗口不在前台时。

失败策略是保守处理：如果不能确认当前区域就是推荐内容，隐藏遮罩，不强行覆盖。

## 启动与单实例

入口文件是 `Start-BiliClientMask.ps1`，批处理 `Start-BiliClientMask.bat` 只负责用隐藏窗口启动它。

启动后脚本会创建一个命名互斥量：

```text
Local\BiliClientMask
```

如果互斥量已经存在，说明工具已经在运行，新的实例直接退出，避免出现多个遮罩互相覆盖。

快捷开关通过命名事件通信：

```text
Local\BiliClientMaskToggle
```

`Toggle-BiliClientMask.ps1` 的逻辑是：

- 如果遮罩进程没运行，先启动遮罩进程。
- 如果遮罩进程已运行，发送 `Local\BiliClientMaskToggle` 事件，让主进程切换启停状态。

开机自启由 `Install-Startup.ps1` 创建当前用户启动文件夹里的快捷方式：

```text
BiliClientMask.lnk
```

取消自启由 `Uninstall-Startup.ps1` 删除这个快捷方式。

## 窗口识别

工具每 120ms 执行一次 `Update-Mask`。

识别前台窗口时先读取：

- Win32 前台窗口句柄。
- 窗口标题。
- 进程名。
- DWM 扩展窗口边界。

B 站客户端窗口识别条件是：

- 进程名含有 `bili`，或进程名包含中文“哔哩哔哩”。
- 或标题含有 `bilibili`、`哔哩哔哩`、`B站`，但排除 Chrome、Edge、Firefox 等浏览器进程。

不能只靠标题判断页面类型。B 站 Windows 客户端很多状态都会使用泛标题，例如首页、搜索、历史记录、个人页、视频页可能都表现为同一个 B 站窗口标题。因此当前逻辑是标题判断加视觉探测。

## 页面模式判断

`Get-TargetWindow` 把当前 B 站窗口粗分成两类：

- `Home`：标题像首页或主入口，例如 `bilibili`、`哔哩哔哩`、`首页`、`推荐`、`直播`、`热门`、`追番`、`影视`。
- `Video`：标题不像首页，也不包含搜索提示时，暂按视频页候选处理。

之后再用更细的探测决定是否真的显示遮罩。

## 首页遮罩逻辑

首页遮罩的默认区域：

```powershell
$CoverLeft = 80
$CoverTop = 96
$CoverRight = 0
$CoverBottom = 0
```

也就是从 B 站窗口左侧导航和顶部栏之后开始，用白色遮罩覆盖推荐流。

首页遮罩显示前会先排除：

- 当前窗口不是 B 站前台窗口。
- 当前窗口全屏。
- 搜索框处于激活状态。
- 当前窗口虽然被标题粗判为首页，但顶部深色栏和右侧深色标签栏显示它其实是视频页。
- 左侧导航显示当前选中的不是“首页”。

左侧导航识别由 `Test-NonHomeLeftNavSelected` 完成。它截取窗口左上区域，寻找 B 站粉色选中态像素。如果粉色选中区域的垂直中心明显低于首页按钮区域，就判定当前是“精选”“动态”“我的”等非首页页面，隐藏遮罩。

这一步是为了解决历史记录、个人页等页面被误认为首页的问题。

## 搜索状态识别

搜索识别先做顶部搜索框视觉探测，再用 UI Automation 读取控件语义。

视觉探测由 `Test-HeaderSearchVisualActive` 完成。它截取窗口顶部中间区域，寻找 B 站搜索框激活时出现的长条粉色边框。这个兜底用于处理客户端没有稳定暴露搜索框或搜索词的情况。

`Test-BilibiliSearchActive` 会查找窗口顶部区域里的 Edit 控件或疑似搜索控件，并判断：

- 搜索框是否有键盘焦点。
- 搜索框里是否有用户输入的有效搜索词。
- 顶部搜索区域是否出现有效文本。

这些文本会被忽略，不算搜索词：

- `bilibili`
- `哔哩哔哩`
- `B站`
- `首页`
- `推荐`
- `直播`
- `热门`
- `追番`
- `影视`
- `精选`
- `动态`
- `我的`
- 搜索框默认占位文本

一旦确认搜索状态，会短暂保持 2000ms，避免页面跳转时遮罩闪现。

搜索结果页不遮挡，因为这是用户主动输入关键词后要求看的内容，不属于首页平台推荐流。

## 视频页右侧推荐遮罩逻辑

泛 B 站标题不能直接套首页遮罩。`Test-BilibiliVideoPageVisual` 会先检查顶部深色栏、播放器顶边和右侧深色标签栏；如果视觉上是视频页，就改走视频页推荐遮罩逻辑，绝不应用首页白色大遮罩。

视频页不再使用固定矩形硬盖右侧栏。

当前配置：

```powershell
$VideoRecommendMaskWidth = 460
$VideoRecommendProbeStartTop = 300
$VideoRecommendProbeBottomMargin = 160
```

流程：

1. 从右侧栏候选区域开始截图取样。
2. 跳过顶部创作者信息、互动按钮、简介、视频选集等区域。
3. 寻找“推荐视频缩略图”样式的图片区块。
4. 找到后，从该缩略图区块上方少量偏移处开始，用深色遮罩覆盖到窗口底部。
5. 找不到缩略图时不显示遮罩。

`Test-ImageLikeRecommendationBlock` 的判断依据是缩略图区域内是否存在足够多、分布足够宽的非暗色像素。视频选集通常是深色文字列表或课程条目，不应被识别成推荐缩略图。

为避免遮罩自己参与截图识别，右侧推荐区域一旦识别成功，会按窗口句柄、窗口尺寸和标题缓存结果。遮罩已经显示时复用缓存，不周期性隐藏再截图。

## 全屏处理

`Test-FullscreenWindow` 会比较 B 站窗口边界和当前屏幕边界，允许 4px 误差。

如果窗口接近覆盖整个屏幕，就判定为全屏状态，隐藏所有遮罩。

这用于处理视频全屏播放：用户此时主要是在看视频内容，不应遮挡。

## 临时关闭与自动恢复

快捷键：

- `Ctrl+Alt+B`
- `Ctrl+Alt+M`
- `Ctrl+Shift+F12`

托盘菜单也能开关遮罩。

如果在首页按快捷键关闭遮罩，脚本会记录：

```powershell
$script:RestoreHomeMaskAfterLeavingHome = $true
```

之后只要用户离开首页，例如进入视频页或搜索页，再回到首页，遮罩会自动恢复。

如果 B 站客户端窗口全部关闭，当前运行期内的临时关闭状态也会恢复为开启。下次开机默认开启，不再通过 `.disabled` 文件持久保存关闭状态。

## 遮罩窗口

遮罩使用 WinForms 窗口 `NoActivateMaskForm`：

- 无边框。
- 不显示在任务栏。
- `TopMost`。
- `WS_EX_NOACTIVATE`，显示时不抢焦点。
- `WS_EX_TOOLWINDOW`，减少任务栏和 Alt-Tab 干扰。

遮罩通过 `SetWindowPos` 定位，并使用 `SWP_NOACTIVATE`，避免打断 B 站客户端的焦点。

首页遮罩颜色是白色：

```powershell
$HomeMaskColor = [System.Drawing.Color]::White
```

视频页右侧推荐遮罩颜色接近 B 站深色侧栏：

```powershell
$VideoRecommendMaskColor = [System.Drawing.Color]::FromArgb(22, 23, 25)
```

## 错误保护

定时器入口包了一层 `try/catch`。

如果某次窗口探测、截图或 UI Automation 抛异常：

1. 把异常写入 `BiliClientMask.log`。
2. 隐藏遮罩。
3. 保持进程继续运行。

这样可以避免一次异常导致后台进程无声退出。

## 网页版扩展逻辑

网页版由 `manifest.json`、`content.js`、`content.css`、`popup.html`、`popup.js` 组成。

扩展只匹配：

```text
https://www.bilibili.com/*
```

`content.js` 只在以下路径启用首页遮罩：

```text
/
/index.html
```

它会监听：

- `DOMContentLoaded`
- `load`
- `resize`
- DOM MutationObserver
- `history.pushState`
- `history.replaceState`
- `popstate`

这样可以适配 B 站网页端的单页应用跳转。

遮罩元素是：

```text
#bili-focus-cover
```

CSS 还会隐藏常见首页推荐卡片类名：

- `.feed-card`
- `.bili-video-card`
- `.bili-feed-card`
- `.recommended-container_floor-aside`
- `.recommended-swipe`
- `.recommend-list-v1`
- `.floor-single-card`

网页端扩展当前只解决网页首页推荐流，不处理 Windows 客户端视频页右侧推荐。

## 已知边界

- B 站客户端不是传统 Win32 原生控件，UI Automation 暴露的信息不稳定，所以不能完全依赖控件树。
- 视频页右侧栏布局会随窗口宽度、视频类型、合集、选集、课程、直播回放等变化，必须继续按视觉区域探测。
- 如果推荐缩略图被懒加载为空白，工具可能暂时不显示右侧推荐遮罩，这是故意保守处理。
- 如果 B 站改版颜色、导航选中态或推荐卡片样式，左侧导航和右侧缩略图探测阈值可能需要调整。
